Java switch statement

Switch case statement is used when we have number of options (or choices) and we may need to perform a different task for each choice.

Its syntax is given below:

switch(expression){
case value1:
 //code bock
 //break;
case value2:
 //code bock
 //break;
default:
it executes when no case is matched
}

Example

The example below uses the weekday number to calculate the weekday name: 
int day = 4;
 switch (day) {
  case 1:
   System.out.println("Monday");
   break;
  case 2:
   System.out.println("Tuesday");
   break;
  case 3:
   System.out.println("Wednesday");
   break;
  case 4:
   System.out.println("Thursday");
   break;
  case 5:
   System.out.println("Friday");
   break;
  case 6:
   System.out.println("Saturday");
   break;
  case 7:
   System.out.println("Sunday");
   break;
}
Output:Thursday

The break Keyword

When Java reaches a break keyword, it breaks out of the switch block.This will stop the execution of more code and case testing inside the block.

Note:If there is no break keyword then all the cases will be excecuted until break encounters

The default Keyword

The default keyword specifies some code to run if there is no case match:

int day = 4;
 switch (day) {
  case 5:
   System.out.println("Monday");
   break;
  case 6:
   System.out.println("Tuesday");
   break;
  case 7:
   System.out.println("Wednesday");
   break;
  default:
   System.out.println("in default");
}
Output:in default

Few points about switch statement

  • Case doesn’t always need to have order 1, 2, 3 and so on. It can have any integer value after case keyword. Also, case doesn’t need to be in an ascending order always, you can specify them in any order based on the requirement.
  • The expression given inside switch should result in a constant value otherwise it would not be valid.

    Valid expression

    switch(1+2+23)
switch(1*2+3%4)

    Invalid expression

    switch(ab+cd)
switch(a+b+c)
  • Nesting of switch statements are allowed, which means you can have switch statements inside another switch. However nested switch statements should be avoided as it makes program more complex and less readable.
